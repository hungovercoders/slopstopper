Practices & Feedback

The categories of automated checks that keep your repo healthy

🔒 Security

  • SAST — Static application security testing flags vulnerabilities in source code before merge
  • DAST — Dynamic scanning probes a running site for common web vulnerabilities
  • Secrets Detection — Scans every commit for accidentally committed credentials or tokens
  • Dependency Scanning — Checks all dependencies for known CVEs on every pull request

🧹 Hygiene

  • Complexity Checks — Enforces complexity thresholds to keep functions readable and maintainable
  • Doc Structure — Validates that required documentation sections are present
  • Doc Accuracy — Ensures documentation reflects the actual state of the codebase
  • Doc Size — Prevents documentation from growing unwieldy or stale

✅ Reliability

  • E2E Tests — Playwright tests verify all pages and user flows work on every pull request
  • Smoke Tests — Lightweight smoke tests run against staging and production after every deploy
  • Accessibility — axe-core audits every page against WCAG 2.1 AA on PRs, after deploys, and on a daily schedule
  • Core Web Vitals — Lighthouse CI measures LCP, INP and CLS on every PR and nightly against production

🤖 Operational

  • Auto PR Labelling — Pull requests are labelled automatically from the paths changed
  • Doc Auto-Updater — A weekly agentic workflow opens PRs to keep documentation in sync with the code
  • Failure Alerts — Failed workflow runs on main raise (or update) a tracking issue automatically

🚀 Deployment

  • Preview Deploys — Every pull request gets a unique preview URL on Netlify for review
  • Production Deploy — Merges to main trigger an automatic production release
  • Preview Cleanup — Stale preview environments are automatically removed after PRs close
