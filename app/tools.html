📋 Task

A Taskfile.yml defines the single source of truth for build, test, lint and scan commands so developers, AI agents and CI all run the same thing.

Taskfile excerpt 
version: '3'
tasks:
  contributing:build:    { cmds: ['npm run build'] }
  contributing:run:      { cmds: ['node server.js'] }
  contributing:test:     { cmds: ['npx playwright test --config=.ss/playwright.config.js'] }
  reliability:accessibility:
    cmds: ['npx playwright test --config=.ss/playwright.config.js .ss/tests/accessibility.spec.ts']

Run from your repo with task ss:contributing:build, task ss:reliability:accessibility, etc.

🎭 Playwright

Portable smoke and accessibility specs live under .ss/tests/ with their own .ss/playwright.config.js, so they never collide with your repo's own tests/ directory.

Learn more about Reliability →
.ss/playwright.config.js 
export default defineConfig({
  testDir: './tests',                 // resolves to .ss/tests
  use: {
    baseURL: process.env.SMOKE_TEST_URL || 'http://localhost:8080',
  },
  projects: [{ name: 'chromium', use: { ...devices['Desktop Chrome'] } }],
});

🔍 Semgrep

Fast static application security testing (SAST). Scans source code for vulnerability patterns and OWASP-style issues before code is merged.

Workflow excerpt 
jobs:
  sast:
    name: Static Application Security Testing with Semgrep
    steps:

🛡️ Trivy

Scans project dependencies for known CVEs on every pull request and push, with severity-based gating to prevent vulnerable packages from reaching production.

    name: Scan Dependencies with Trivy

🔑 Gitleaks

Inspects every commit for accidentally committed credentials, tokens and other secrets, blocking pull requests when high-confidence matches are found.

Workflow excerpt 
jobs:
  secrets:
    name: Detect Secrets with Gitleaks
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - uses: gitleaks/gitleaks-action@v2
